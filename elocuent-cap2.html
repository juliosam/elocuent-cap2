Eensayo y ejercicios del capitulo 2 de Elocuent JavaScript

Este capitulo nos habla de como una expresion de javascript puede ser desde un valor como un numero o string, hasta un valor entre parentesis (extenso). Compara a las expresiones con sub-oraciones y que de la misma manera que estas, una expresion puede ser formada de otras expresiones. Y si una expresion es una sub-oracion entonces una declaracion de javascript seria una oracion completa y un programa es una lista de declaraciones.

Tambien se nos describe a las variables, que sirven para guardar una expresion para su posterior uso. Y que estas se comportan mas como tentaculos que "agarran" un valor o expresion, mas que como una caja que las guarda.Por esto mismo las variables (excepto const) pueden soltar un valor y tomar otro , si asi se declara mas adelante en el programa.

Las variables se pueden nombrar de muchas maneras pero con algunas excepciones. No pueden iniciar con un digito y solo puden contener los simbolos "$" y "_", ningun otro simbolo ni caracter especial. Ademas existe una lista de nombres reservados para el uso de javascript, pero no se necesitan recordar, ya que al momento de intentar utilizar uno se produce un error en automatico.

Las funciones son partes de programa guardadas en una variable, que al invocarlas (llamarlas) te muestran el resultado del programa que contiene, con respecto a los parametros que se les marquen (o la ausencia de ellos). Un resultado es cuando regresan un valor, aunque a veces las funciones son utiles por los efectos laterales que producen (como mandar texto a la pantalla).

Los programas pueden tener diferentes controles de flujo, hay lineales que solo dan el resultado, hay otros con condicionantes como lo son if, else y else if, y otros que trabajan en loops, ademas de los que combinan a estos.
